# AI Ora Hub — прототип платформы для рекрутинга

Демо платформы для международного найма: путь работодателя и специалиста через заявку, предварительную оценку, AI-интервью, оффер, релокацию и визовое сопровождение.

Продуктовое назначение, текущие возможности и правила интерфейса зафиксированы в [`docs/product-brief.md`](docs/product-brief.md).

## Стек

Java 17+, Spring Boot 3.2 (Web, Data JPA, Thymeleaf), PostgreSQL в production, Flyway, файловая H2 для локальной разработки, собственная дизайн-система AI Ora Hub и Tailwind CSS через CDN на служебных экранах.

## Запуск

```bash
mvn spring-boot:run
```

Откройте http://localhost:8080

Для локального запуска внешняя БД не нужна: данные сохраняются в `data/aiorahub.mv.db`, а Flyway создаёт схему. Mock-данные загружаются только в полностью пустую БД (работодатель с тремя вакансиями и кандидат Anna, уже прошедшая AI-интервью).

Чтобы начать локально с пустой БД, остановите приложение и удалите каталог `data/`.

## Демо-сценарий

1. На главной нажмите «Войти как Работодатель» — в разделе «Отклики» уже виден кандидат Anna с отчётом AI (уровень языка B2, комментарий об адекватности). Нажмите «Выдать Job Offer» — или сначала создайте свою вакансию через форму.
2. Выйдите и войдите как Соискатель. Откликнитесь на вакансию → кнопка «Пройти AI-интервью».
3. На экране интервью: «Подключиться через Google Meet» (открывает реальный Meet) и «Завершить интервью» — система сгенерирует фейковый отчёт AI и передаст его работодателю.
4. Снова войдите как Работодатель и выдайте Job Offer вашему кандидату.
5. Вернитесь как Соискатель: появилась кнопка визового модуля. Загрузите три документа (паспорт, диплом, контракт — файлы не сохраняются, только метаданные) и продвигайте процесс демо-кнопкой: Документы → Проверка платформой → Посольство Германии → Виза одобрена.

Авторизация упрощена намеренно: вход одной кнопкой под демо-пользователем нужной роли, id хранится в HTTP-сессии. Spring Security не используется.

## Production

- Домен: https://aiorahub.com
- Сервер: DigitalOcean Droplet, Ubuntu, Nginx -> Spring Boot на `127.0.0.1:8080`.
- Runtime: отдельный системный пользователь `aiorahub`, PostgreSQL доступна только локально.
- DNS управляется Cloudflare.
- Инструкции по сборке, выкладке, rollback и проверке: [`docs/production-runbook.md`](docs/production-runbook.md).

Production остаётся демо-средой по способу авторизации и интеграциям, но данные сохраняются в PostgreSQL между перезапусками.

## Структура

```
src/main/java/com/truehire/
├── TrueHireApplication.java
├── config/DataInitializer.java        # mock-данные при старте
├── controller/
│   ├── AuthController.java            # вход/выход по ролям
│   ├── EmployerController.java        # вакансии, отклики, Job Offer
│   └── CandidateController.java       # отклики, AI-интервью, виза
├── model/                             # User, JobVacancy, JobApplication,
│                                      # InterviewResult, VisaDocument + enums
└── repository/                        # Spring Data JPA репозитории

src/main/resources/templates/          # index, employer, candidate, interview, visa
```
