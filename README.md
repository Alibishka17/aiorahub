# AI Ora Hub — прототип платформы для рекрутинга

Демо платформы для международного найма: путь работодателя и специалиста через заявку, предварительную оценку, AI-интервью, оффер, релокацию и визовое сопровождение.

Продуктовое назначение, текущие возможности и правила интерфейса зафиксированы в [`docs/product-brief.md`](docs/product-brief.md).

## Стек

Java 17+, Spring Boot 3.2 (Web, Security, Data JPA, Thymeleaf), PostgreSQL в production, Flyway, файловая H2 для локальной разработки, BCrypt, защищённые HTTP-сессии и собственная дизайн-система AI Ora Hub.

## Запуск

```bash
mvn spring-boot:run
```

Откройте http://localhost:8080

Для локального запуска внешняя БД не нужна: данные сохраняются в `data/aiorahub.mv.db`, а Flyway создаёт схему. Mock-данные загружаются только в полностью пустую БД (работодатель с тремя вакансиями и кандидат Anna, уже прошедшая AI-интервью).

Чтобы начать локально с пустой БД, остановите приложение и удалите каталог `data/`.

## Рабочий сценарий

1. Создайте аккаунт работодателя или соискателя. Регистрация требует имя, фамилию, международный номер, почту и пароль; Telegram и WhatsApp отмечаются для указанного номера.
2. Работодатель создаёт вакансии и управляет состояниями «Опубликована», «Скрыта» и «Удалена из рабочих списков» без потери истории откликов.
3. Соискатель загружает CV, выбирает опубликованную вакансию и проходит демонстрационное AI-интервью.
4. Работодатель видит контакты, CV, summary, транскрибацию и заключение, затем подтверждает кандидата с персональным письмом.
5. Соискатель получает письмо в истории отклика и может перейти к демонстрационному визовому процессу.

Пароли хранятся только как BCrypt-хеши. Доступ к кабинетам и CV проверяется по роли и владельцу данных; изменяющие формы защищены CSRF-токеном. Production-сессия передаётся только cookie с `Secure`, `HttpOnly` и `SameSite=Lax`.

## Production

- Домен: https://aiorahub.com
- Сервер: DigitalOcean Droplet, Ubuntu, Nginx -> Spring Boot на `127.0.0.1:8080`.
- Runtime: отдельный системный пользователь `aiorahub`, PostgreSQL доступна только локально.
- DNS управляется Cloudflare.
- Инструкции по сборке, выкладке, rollback и проверке: [`docs/production-runbook.md`](docs/production-runbook.md).

Данные аккаунтов и процесса найма сохраняются в PostgreSQL, а CV — в `/var/lib/aiorahub/uploads`. AI-интервью и визовый процесс пока остаются демонстрационными интеграциями.

## Структура

```
src/main/java/com/truehire/
├── TrueHireApplication.java
├── config/                            # security и mock-данные пустой БД
├── controller/
│   ├── AuthController.java            # регистрация, вход и выход
│   ├── EmployerController.java        # вакансии, отклики, отчёты и подтверждения
│   └── CandidateController.java       # CV, отклики, AI-интервью и виза
├── model/                             # User, JobVacancy, JobApplication,
│                                      # InterviewResult, VisaDocument + enums
├── repository/                        # Spring Data JPA репозитории
└── service/                           # BCrypt и защищённое хранение CV

src/main/resources/templates/          # публичная страница, auth и кабинеты
```
