# AIOraHub — платформа для рекрутинга

Платформа для международного найма: путь работодателя и специалиста через заявку, предварительную оценку, AI-интервью, оффер, релокацию и визовое сопровождение.

Продуктовое назначение, текущие возможности и правила интерфейса зафиксированы в [`docs/product-brief.md`](docs/product-brief.md).

## Стек

Java 17+, Spring Boot 3.2 (Web, Security, Data JPA, Thymeleaf), PostgreSQL в production, Flyway, файловая H2 для локальной разработки, BCrypt, защищённые HTTP-сессии и собственная дизайн-система AIOraHub.

Интерфейс доступен на русском, казахском и английском. При первом открытии язык выбирается по `Accept-Language` браузера (`ru`, `kk`, `en`), неизвестные языки получают английский. Ручной выбор сохраняется на год в cookie `AIORAHUB_LANG`. Тексты вакансий и сообщения рекрутера показываются на языке, на котором их ввёл пользователь.

## Запуск

```bash
mvn spring-boot:run
```

Откройте http://localhost:8080

Для локального запуска внешняя БД не нужна: данные сохраняются в `data/aiorahub.mv.db`, а Flyway создаёт пустую схему. Тестовые пользователи, вакансии и результаты автоматически не создаются.

Чтобы начать локально с пустой БД, остановите приложение и удалите каталог `data/`.

## Рабочий сценарий

1. Открытые вакансии доступны без входа. Гость может оставить имя, фамилию, международный номер, Telegram/WhatsApp и обязательную почту без создания аккаунта.
2. Для постоянной работы создайте аккаунт работодателя или соискателя. Регистрация требует имя, фамилию, международный номер, почту и пароль; Telegram и WhatsApp отмечаются для указанного номера.
3. Работодатель создаёт вакансии со страной, городом, направлением, диапазоном зарплаты, документами для работы и дополнительными условиями. В списке видны состояние, просмотры и отклики; форма создания открывается только по команде «Создать вакансию».
4. Зарегистрированный соискатель ищет вакансии по тексту, направлению, стране, городу и зарплате, загружает CV, открывает полную страницу позиции и отправляет отклик без повторного ввода контактов.
5. Для каждой новой вакансии работодатель или администратор настраивает отдельное AI-интервью: RU/EN, 1–8 вопросов, критерии и уровни, intro/outro, режим конструктора или дополнительного полного prompt, а также разрешённую свободу аватара. Без обязательной настройки вакансия не публикуется.
6. При создании отклика конфигурация интервью и все данные вакансии фиксируются в неизменяемом snapshot. Последующие изменения вакансии действуют только на новые отклики. Существующие demo-вакансии без конфигурации продолжают использовать закреплённый HRme template.
7. Каждый отклик получает отдельную HRme-сессию. Интервью ограничено пятью минутами; структурированный summary формируется на языке кабинета работодателя (RU/KK) и содержит баллы, аргументацию, сильные стороны, риски и рекомендацию.
8. Работодатель видит контакты и CV зарегистрированного пользователя. Summary и транскрибация показываются только для результатов, полученных от HRme. Гостевой отклик помечен «Без регистрации» и содержит действие для ручного письма на email.
7. Зарегистрированный соискатель получает решение в истории отклика и может перейти к визовому процессу. Для гостевого отклика рекрутер связывается с кандидатом самостоятельно; автоматическая почтовая отправка пока не подключена.

Кабинет работодателя разделён на рабочие экраны `Дашборд`, `Вакансии`, `Кандидаты`, `Настройки`. Кабинет соискателя содержит `Дашборд`, `Вакансии`, `Мои отклики`, `Документы`, `Настройки`. AI-отчёт и транскрибация находятся внутри записи кандидата/отклика, а не в отдельном дублирующем разделе. В настройках пользователь может менять профиль, контакты, email и пароль; смена пароля требует текущий пароль.

Служебная панель `/admin` защищена отдельной BCrypt-аутентификацией. Она показывает first-party статистику без сохранения исходных IP-адресов, управляет вакансиями и предоставляет тот же редактор AI-интервью, что и кабинет работодателя.

Пароли хранятся только как BCrypt-хеши. Доступ к кабинетам и CV проверяется по роли и владельцу данных; изменяющие формы защищены CSRF-токеном. Сессия хранится в JDBC-хранилище 24 часа, а production cookie имеет `Secure`, `HttpOnly`, `SameSite=Lax` и `Max-Age=24h`, поэтому вход сохраняется при переходах, закрытии браузера и рестарте приложения.

## Production

- Домен: https://aiorahub.com
- Сервер: DigitalOcean Droplet, Ubuntu, Nginx -> Spring Boot на `127.0.0.1:8080`.
- Runtime: отдельный системный пользователь `aiorahub`, PostgreSQL доступна только локально.
- DNS управляется Cloudflare.
- Инструкции по сборке, выкладке, rollback и проверке: [`docs/production-runbook.md`](docs/production-runbook.md).

Данные аккаунтов, сессий, аналитики и процесса найма сохраняются в PostgreSQL, а CV — в `/var/lib/aiorahub/uploads/cv`. Systemd задаёт `UPLOAD_DIR=/var/lib/aiorahub/uploads`; этот путь должен входить в `ReadWritePaths` и принадлежать пользователю `aiorahub`. AI-интервью запускаются через защищённый service-to-service API HRme; AIOraHub не создаёт фиктивные результаты. Защищённое хранилище визовых документов подключается отдельной интеграцией.

## Структура

```
src/main/java/com/truehire/
├── TrueHireApplication.java
├── config/                            # security и runtime-конфигурация
├── controller/
│   ├── AuthController.java            # регистрация, вход и выход
│   ├── PublicVacancyController.java   # публичные вакансии и гостевые отклики
│   ├── EmployerController.java        # вакансии, отклики, отчёты и подтверждения
│   ├── CandidateController.java       # CV, отклики и виза
│   ├── AdminController.java           # статистика и управление вакансиями
│   └── IntegrationController.java     # защищённый контракт с HRme
├── model/                             # User, JobVacancy, JobApplication,
│                                      # InterviewResult, VisaDocument + enums
├── repository/                        # Spring Data JPA репозитории
└── service/                           # BCrypt, настройки аккаунта и защищённое хранение CV

src/main/resources/templates/          # публичная страница, auth и кабинеты
```
