# AI Ora Hub — платформа для рекрутинга

Платформа для международного найма: путь работодателя и специалиста через заявку, предварительную оценку, AI-интервью, оффер, релокацию и визовое сопровождение.

Продуктовое назначение, текущие возможности и правила интерфейса зафиксированы в [`docs/product-brief.md`](docs/product-brief.md).

## Стек

Java 17+, Spring Boot 3.2 (Web, Security, Data JPA, Thymeleaf), PostgreSQL в production, Flyway, файловая H2 для локальной разработки, BCrypt, защищённые HTTP-сессии и собственная дизайн-система AI Ora Hub.

Интерфейс доступен на русском, казахском и английском. При первом открытии язык выбирается по `Accept-Language` браузера (`ru`, `kk`, `en`), неизвестные языки получают английский. Ручной выбор сохраняется на год в cookie `AIORAHUB_LANG`. Тексты вакансий и сообщения рекрутера показываются на языке, на котором их ввёл пользователь.

## Запуск

```bash
mvn spring-boot:run
```

Откройте http://localhost:8080

Для локального запуска внешняя БД не нужна: данные сохраняются в `data/aiorahub.mv.db`, а Flyway создаёт пустую схему. Тестовые пользователи, вакансии и результаты автоматически не создаются.

Чтобы начать локально с пустой БД, остановите приложение и удалите каталог `data/`.

## Рабочий сценарий

1. Открытые вакансии доступны без входа. Гость может оставить имя, фамилию, международный номер, Telegram/WhatsApp и обязательную почту без создания аккаунта.
2. Для постоянной работы создайте аккаунт работодателя или соискателя. Регистрация требует имя, фамилию, международный номер, почту и пароль; Telegram и WhatsApp отмечаются для указанного номера.
3. Работодатель создаёт вакансии и управляет состояниями «Опубликована», «Скрыта» и «Удалена из рабочих списков» без потери истории откликов.
4. Зарегистрированный соискатель загружает CV, открывает полную страницу опубликованной вакансии и отправляет отклик без повторного ввода контактов.
5. Каждый отклик получает отдельную HRme-сессию по шаблону вакансии. После завершения HRme возвращает summary и транскрибацию в тот же отклик; повторный запуск использует уже созданную сессию.
6. Работодатель видит контакты и CV зарегистрированного пользователя. Summary, транскрибация и заключение показываются только для результатов, полученных от HRme. Гостевой отклик помечен «Без регистрации» и содержит действие для ручного письма на email.
7. Зарегистрированный соискатель получает решение в истории отклика и может перейти к визовому процессу. Для гостевого отклика рекрутер связывается с кандидатом самостоятельно; автоматическая почтовая отправка пока не подключена.

Служебная панель `/admin` защищена отдельной BCrypt-аутентификацией. Она показывает first-party статистику без сохранения исходных IP-адресов и управляет вакансиями, их публикацией и HRme template ID.

Пароли хранятся только как BCrypt-хеши. Доступ к кабинетам и CV проверяется по роли и владельцу данных; изменяющие формы защищены CSRF-токеном. Production-сессия передаётся только cookie с `Secure`, `HttpOnly` и `SameSite=Lax`.

## Production

- Домен: https://aiorahub.com
- Сервер: DigitalOcean Droplet, Ubuntu, Nginx -> Spring Boot на `127.0.0.1:8080`.
- Runtime: отдельный системный пользователь `aiorahub`, PostgreSQL доступна только локально.
- DNS управляется Cloudflare.
- Инструкции по сборке, выкладке, rollback и проверке: [`docs/production-runbook.md`](docs/production-runbook.md).

Данные аккаунтов, аналитики и процесса найма сохраняются в PostgreSQL, а CV — в `/var/lib/aiorahub/uploads`. AI-интервью запускаются через защищённый service-to-service API HRme; AioraHub не создаёт фиктивные результаты. Защищённое хранилище визовых документов подключается отдельной интеграцией.

## Структура

```
src/main/java/com/truehire/
├── TrueHireApplication.java
├── config/                            # security и runtime-конфигурация
├── controller/
│   ├── AuthController.java            # регистрация, вход и выход
│   ├── PublicVacancyController.java   # публичные вакансии и гостевые отклики
│   ├── EmployerController.java        # вакансии, отклики, отчёты и подтверждения
│   ├── CandidateController.java       # CV, отклики и виза
│   ├── AdminController.java           # статистика и управление вакансиями
│   └── IntegrationController.java     # защищённый контракт с HRme
├── model/                             # User, JobVacancy, JobApplication,
│                                      # InterviewResult, VisaDocument + enums
├── repository/                        # Spring Data JPA репозитории
└── service/                           # BCrypt и защищённое хранение CV

src/main/resources/templates/          # публичная страница, auth и кабинеты
```
